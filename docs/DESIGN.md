# Room Management System

## System Architecture and Design

### Overview
The Room Management System (RMS) is designed to efficiently manage rooms, bookings, and associated equipment within an educational institution. The architecture will provide a clear separation of concerns, enhancing maintainability and scalability.

### Key Components
- **User Roles**:
  1. Admin
  2. BGH (Board of Education)
  3. GV Manager (General Manager)
  4. GV Staff (General Staff)

- **Room Management**: Management of 7 different rooms, including their characteristics and configurations.
- **Equipment Management**: Inventory management for all equipment associated with the rooms.
- **Booking System**: A robust system to handle bookings, cancellations, and modifications.
- **Statistics/Reporting Features**: Reporting tools to analyze usage and performance metrics.

### Architectural Diagram
- A visual representation will be included in the final documentation.

## Design Considerations
- Scalability: Ability to add new user roles or rooms without major system rework.
- Security: Role-based access control.

## Technologies Used
- Frontend: React.js
- Backend: Node.js, Express
- Database: MongoDB/PostgreSQL

## Deployment
- Deployed on AWS using Docker containers.

## Conclusion
The Room Management System is designed to improve the management efficiency and enhance user experience in room booking and management.
